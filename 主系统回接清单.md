## 3. 第一阶段 API 契约

第一阶段只实现定位接口，不生成项目包。

```text
POST /api/event-graph/ideas/position
POST /api/event-graph/ideas/{idea_id}/generate-project-draft
```

请求体建议：

```json
{
  "idea_text": "我想做一个可以用语音控制的送货小车",
  "age": 8,
  "age_band": null,
  "current_project_id": null,
  "known_knowledge_point_ids": [],
  "available_hardware": ["Arduino", "超声波传感器", "小车底盘"],
  "duration_hours": 8,
  "target_use": "classroom",
  "teacher_constraints": []
}
```

响应体必须包含：

- `stage_decision`：推荐年龄层、插入类型、置信度和理由。
- `knowledge_chain`：前置、核心、后续知识点。
- `insertion_slots`：候选插入位、锚定项目关键词和评分。
- `candidate_existing_project_keywords`：候选项目关键词。
- `generation_recommendation`：复用、变体或新项目包建议。
- `risk_flags`：年龄风险、硬件未知、AI 过度依赖等风险。

第二阶段 `generate-project-draft` 响应必须包含：

- `status=draft`，且 `project_manifest.status=draft`。
- `files` 覆盖 `project_plugin_service.REQUIRED_PACKAGE_FILES` 中所有必备文件。
- `project_manifest` 覆盖 `project_plugin_service.REQUIRED_PROJECT_FIELDS` 中所有必填字段。
- `package_files` 包含 `project.json`、`curriculum.md`、`lesson_plan.json`、`knowledge_map.json`、`materials.json`、`assessment.json`、`ai_guardrails.json`、`zongping_map.json`、`competition_map.json`、`parent_view.md`、`teacher_notes.md`、`README.md`。
- `validation_preview.can_activate=false`，并列出教师确认、教研审核和项目插件校验。
- L1 竞赛想法仍保持前置补齐，不生成 `target_competitions`，不进入竞赛包装。
