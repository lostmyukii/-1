# 域名接入执行清单

> 更新时间：2026-05-30 22:10 CST
> 目标：为事理图谱教师端与管理端补齐 `vsle.ilelezhan.com` 入口。
> 当前可用正式入口：`http://vsle.ilelezhan.cn`。

## 当前结论

- `vsle.ilelezhan.cn` 已可用，权威 DNS 返回 `49.232.81.132`。
- `ilelezhan.com` 当前在 `.com` 注册局层没有可查询的注册/委派记录。
- `vsle.ilelezhan.com` 当前无法通过 DNS 解析，不是主系统代码、服务器服务、Caddy/Nginx 或事理图谱审核链问题。
- 在 `.com` 域名完成注册/找回和 NS 委派前，不应继续修改主系统代码或重启服务器来排查该问题。

## 已验证证据

```bash
dig +short NS ilelezhan.com
dig +short SOA ilelezhan.com
dig @a.gtld-servers.net ilelezhan.com NS +noall +answer +authority
dig +trace ilelezhan.com NS
whois ilelezhan.com
```

结果摘要：

- `ilelezhan.com` 未返回 NS 或 SOA。
- `.com` 注册局只返回 `com.` 的 SOA/NSEC3，未返回 `ilelezhan.com` 委派。
- `whois ilelezhan.com` 返回 `No match for domain "ILELEZHAN.COM"`。

对照 `.cn`：

```bash
dig +short NS ilelezhan.cn
dig @example.dnspod.net vsle.ilelezhan.cn A +short
dig @rapper.dnspod.net vsle.ilelezhan.cn A +short
```

结果摘要：

- `ilelezhan.cn` 使用 DNSPod：`example.dnspod.net.`、`rapper.dnspod.net.`。
- `vsle.ilelezhan.cn` 在两个权威服务器上均返回 `49.232.81.132`。

## 外部执行步骤

1. 确认是否确实需要 `.com` 入口。
   - 如果不需要，继续使用 `http://vsle.ilelezhan.cn` 作为正式入口。
   - 如果需要，进入下一步。

2. 确认 `ilelezhan.com` 的域名状态。
   - 在域名注册商控制台搜索 `ilelezhan.com`。
   - 如果未注册，完成注册。
   - 如果已由其他账号持有，先找回或转移到当前可管理账号。

3. 配置 `ilelezhan.com` 的 NS 委派。
   - 推荐与 `.cn` 保持一致，使用 DNSPod 或当前团队统一 DNS 服务。
   - 如果使用 DNSPod，可将 NS 设置为控制台分配的两条权威服务器。
   - 不要把生产密钥、账号密码或控制台截图写入仓库。

4. 添加业务解析记录。
   - 主机记录：`vsle`
   - 记录类型：`A`
   - 记录值：`49.232.81.132`
   - TTL：使用默认值；如需快速验收，可临时设为 600 秒或更低。

## 生效验收

DNS 生效后执行：

```bash
dig +short NS ilelezhan.com
dig +short vsle.ilelezhan.com
curl --noproxy '*' -I http://vsle.ilelezhan.com/teacher/project-package-review
curl --noproxy '*' -I http://vsle.ilelezhan.com/admin/event-graph/project-package-review
curl --noproxy '*' -I http://vsle.ilelezhan.com/api/health
```

期望：

- `dig +short NS ilelezhan.com` 返回已配置的权威 DNS。
- `dig +short vsle.ilelezhan.com` 返回 `49.232.81.132`。
- 教师端和管理端入口返回 200，或管理端在未登录状态下返回正常页面。
- `/api/health` 返回后端健康检查 200。

## 失败处理

- 如果 `ilelezhan.com` 仍无 NS/SOA：检查域名是否完成注册、实名/邮箱验证、NS 是否已保存。
- 如果 `ilelezhan.com` 有 NS 但 `vsle` 无 A 记录：检查 DNS 控制台是否在正确 zone 添加了 `vsle`。
- 如果 `vsle.ilelezhan.com` 已解析但页面不可达：再检查服务器入口、反向代理和服务状态。
- 若未完成 `.com` 域名配置，不进入服务器更新阶段，不修改事理图谱代码。
